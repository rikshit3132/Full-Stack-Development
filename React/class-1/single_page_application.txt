#### Single Page Application

A Single Page Application (SPA) in React means your 
app loads one HTML page once, and after that everything
happens using JavaScript (no full page reloads).

#### What is a Single Page Application?
In a SPA:
-> Browser loads index.html only once
-> Navigation (Home → About → Profile) happens
 without page reload
-> Data is fetched via APIs (AJAX / fetch / axios)
-> UI updates dynamically
-->> Example: Gmail, Google Docs, Facebook, Twitter

### Why NOT traditional multi-page apps?
Traditional apps:
-> Every click → new HTML page
-> Full page reload
-> Slower user experience
-> More server load
-->> Bad UX
-->> slow
-->> Repetitive HTML rendering

#### WHY Single Page Application?
1) Faster user Experience
-> Page loads once
-> Only data changes, not entire page
-> Feels like a desktop application

Note:-
-->> Traditional: Page reload → HTML → CSS → JS
-->> SPA: Fetch data → Update UI

2) Smooth Navigation (No Reload)
-> Clicking links does NOT refresh the page
-> React Router changes URL without reload

-->> <Route path="/profile" element={<Profile />} />

3) Better User Experience (UX)
-> No flicker
-> No white screen between pages
-> Animations & transitions possible

4) Backend & Frontend Separation
-> Frontend (React) → UI
-> Backend (Node / Java / Python) → APIs
-->> React → API → JSON → React
-> Clean architecture
-> Easy to scale
-> Mobile apps can use same APIs


##### Internal flow of SPA

STEP 1: Initial Request
User opens:
https://myapp.com
Browser sends request to server

STEP 2: Server Response (ONLY ONCE)
Server returns:
-->> index.html
-->> bundle.js (React code)
-->> styles.css
<body>
  <div id="root"></div>
  <script src="bundle.js"></script>
</body>

Note:- This is the ONLY full page load.

STEP 3: React Bootstraps the App
React runs:
ReactDOM.createRoot(document.getElementById("root"))
  .render(<App />);

✔ React takes control of the page
✔ Virtual DOM is created
✔ Components are rendered

STEP 4: Initial UI Render
React builds:
-->> Virtual DOM
-->> Compares with real DOM
-->> Updates real DOM
Note:- Screen shows Home Page

STEP 5: User Clicks a Link

User clicks:
-->> /profile
-->> No page reload
-->> No server HTML request

STEP 6: Client-Side Routing
React Router intercepts navigation:
<Route path="/profile" element={<Profile />} />
✔ URL changes
✔ Browser history updated
✔ No reload

STEP 7: Component Loads
React:
-->> Unmounts previous component
-->> Mounts new component (Profile)
Lifecycle:
Unmount Home
Mount Profile

STEP 8: API Call (If Needed)
Profile page needs data:

useEffect(() => {
  fetch("/api/user")
    .then(res => res.json())
    .then(data => setUser(data));
}, []);

✔ Backend sends JSON
✔ React updates state
✔ UI updates

STEP 9: State Change → Re-render
State changes:
setUser(data)
React:
-> Updates Virtual DOM
-> Diffs old vs new
-> Updates ONLY changed parts
--->> Super fast

STEP 10: Repeat (No Reload Ever)
User navigates:
/dashboard
/settings
/logout
✔ Same page
✔ Same JS runtime
✔ Only components & data change

### SPA INTERNAL FLOW (ONE-LINE VIEW)
index.html → React loads → Router handles navigation 
→ API fetch → State change → UI update

Gives files once

Client:
Handles navigation, UI, data, state
